HTML

HTML hvis man skal forklare det meget simpel, er skeletet af en hjemmeside. Det er hvad der holder fast på alt vi ser på vores hjemmeside, og hvor vi placere vores tekst, billeder og links.

Editors

En editor bruges til at skrive og rette i koden. Der findes mange valgmuligheder af editors, som feks. Sumblime, Brackets, PHPStorm, Dreamviewer osv. I sidste ende kunne man også bruge et helt normalt tekst skrivnings program som feks. notepad, eller notepad++. Dog ville det være en dum idé at bruge sådan noget som Microsoft Word, da den vil kigge anderledes på de forskellige tegn i koden.

Struktur & Opsætning

For at kunne starte med at skrive i HTML, skal der være en bestemt struktur og opsætning som forklare HTML hvad der er hvad.
En typisk måde at sætte sin kode op kunne være:


<!DOCTYPE html> Hvad type dokument og kode vi arbejder i.


<html> HTML er åbnet op her.


<head> I vores head kan vi placere vores meta tags, og alt det information der ikke skulle kunne ses på siden.


<title></title> Vores tittel til siden i browseren.


</head> Afslutning af vores head område.


<body> Her begynder vi på body, som er hvor vi kan placere vores kode som kommer til at blive vidst på siden.


</body> Afslutning af vores body.


</html> Slut HTML tag, for at makere afslutning af HTML koden.

Semantiske Tags

Disse tags hjælper med at gøre din hjemmeside mere søgebar (SEO), man kan også sige at tagsene har betydning. Vi bruger Semantiske tags til at forklare en browser hvad der er på siden, og hvor det er. Ved brug af feks. <header> tagget, så ved browseren at det er højst sandsyndlig vores navbar, eller noget der ville være ligende. Her er nogen af dem:
<article>
<aside>
<details>
<figcaption>
<figure>
<footer>
<header>
<main>
<mark>
<nav>
<section>
<summary>
<time>

Meta tags

Metatags handler om data. Metataget giver information omkring HTML dokumentet som ikke vil blive vidst på siden selv. Metatags bliver brugt til at lave specifike side beskrivelser, keywords, forfatter af dokumentet, sidst modificeret osv. Metadata kan blive brugt af browsers til at forklare hvordan siden skal vises, hvilket sprog siden er på osv. Det kan også hjælpe søgemaskiner med at vise din side ved specifike søge ord. Meta tags skal altid være i <head> Eksempler:

<meta name="keywords" centent="HTML, CSS, XML, XHTML, JavaScript">
<meta name="description" content="Min HTML side om multimediedesigner">
<meta name="author" content="Christian Dahl">
<meta http-equiv="refresh" content="30">
<meta name="viewport" content="width=device-width, initial-scale=1.0">

Attributes

Her er nogen eksempler på andre attributes brugt i metatags:
Charset: Fortæller hjemmesiden hvilket sprog der bliver skrevet i på hjemmesiden.
Content: Giver value til http-equiv eller name attributen.
Name: Fortæller os navnet på den bestemte metadata.

Atributes

En atribute er modifier til et HTML element. Vi bruger det til at fortælle vores linje kode hvad den skal gøre eller hvordan den skal opføre sig.
Eksempel:


<tag attribute="value">Noget tekst vi gerne vil modify i vores tag</tag>

Der findes rigtig mange attributes, så for at gøre det nemmere, så hop ind på W3Schools hjemmeside og se selv

Validering

Validering hjælper med at fortælle dig om du har lavet nogen fejl på din side, som vil give problemer længere henne når du feks. kommer til din SEO. Hvis din hjemmeside ikke overholder validerings reglerne, så er der stor chance for at du er bagerst i køen på din søgemaskine.

Du kan valider din hjemmeside inde på W3C's validerings service
